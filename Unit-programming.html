Programming

The foundation of software development. This unit teaches how to think logically, design solutions and communicate instructions to computers using programming languages.

01 · Introduction to Programming

Programming is the process of creating instructions that tell a computer how to perform a task.

A programmer transforms a problem from the real world into a logical solution that a computer can execute.

  • Problem analysis
  • Solution design
  • Writing code
  • Testing and debugging
  • Maintenance

02 · Programming Concepts

Programming is built around fundamental ideas that allow computers to solve problems.

  • Input — information given to a program
  • Processing — operations performed on data
  • Output — results produced by the program
  • Storage — saving information for future use

03 · Problem Solving in Programming

Before writing code, programmers must understand the problem and design a clear solution.

  • Identify the problem
  • Determine inputs and outputs
  • Create an algorithm
  • Convert the algorithm into code
  • Test the solution

04 · Algorithms

An algorithm is a step-by-step procedure used to solve a problem. It describes the logic of a solution before writing actual program code.

A good algorithm should be:

  • Clear: Every step should be understandable.
  • Finite: It must eventually stop.
  • Correct: It should produce the expected result.
  • Efficient: It should use resources effectively.

Example: Algorithm for adding two numbers:

  • Start
  • Input first number
  • Input second number
  • Add the numbers
  • Display the answer
  • Stop

05 · Flowcharts

A flowchart is a visual representation of an algorithm using symbols connected by arrows.

  • Oval: Start and end points
  • Rectangle: Processing steps
  • Parallelogram: Input and output operations
  • Diamond: Decisions and conditions
  • Arrow: Flow direction

Flowcharts help programmers understand program logic before implementation.

06 · Pseudocode

Pseudocode is an informal way of writing program instructions using simple English-like statements.

It allows programmers to focus on logic without worrying about programming language rules.

Example:

  • BEGIN
  • INPUT number
  • IF number is greater than zero
  • DISPLAY "Positive"
  • END IF
  • END

07 · Programming Languages

A programming language is a formal language used to communicate instructions to a computer.

Programming languages are divided into different levels:

  • Machine Language: Instructions written in binary (0s and 1s).
  • Assembly Language: Low-level language using symbolic instructions.
  • High-Level Languages: Human-readable languages such as C, Java, Python and JavaScript.

08 · Introduction to C Programming

C is a powerful general-purpose programming language widely used for system programming, embedded systems and learning programming fundamentals.

A simple C program structure:

  • Preprocessor directives
  • Main function
  • Statements
  • Return statement

Example: 

#include <stdio.h>

int main()
{
    printf("Hello World");
    return 0;
}

09 · Variables and Constants

A variable is a named storage location used to hold data that can change during program execution.

A constant is a value that cannot be changed after it is defined.

  • Variable declaration
  • Variable initialization
  • Assigning values
  • Using variables in calculations

10 · Data Types

Data types define the kind of information a variable can store and how much memory is required.

  • int: Stores whole numbers.
  • float: Stores decimal numbers.
  • double: Stores larger decimal values with more precision.
  • char: Stores a single character.
  • string: Stores a collection of characters.

11 · Operators and Expressions

Operators are symbols used to perform operations on data.

  • Arithmetic operators: +, -, *, /, %
  • Relational operators: >, <, ==, !=
  • Logical operators: AND, OR, NOT
  • Assignment operators: =, +=, -=

Expressions combine variables, values and operators to produce a result.

12 · Input and Output

Input allows a program to receive information from the user, while output displays results.

In C programming:

  • printf(): Used to display output.
  • scanf(): Used to receive input.

13 · Decision Making

Decision structures allow programs to make choices based on conditions.

  • if statement
  • if-else statement
  • else-if ladder
  • switch statement

Example: A program can check whether a student has passed or failed depending on marks.

14 · Loops and Repetition

Loops allow a program to execute a block of code repeatedly until a condition is met.

  • for loop: Used when the number of repetitions is known.
  • while loop: Repeats while a condition remains true.
  • do-while loop: Executes at least once before checking the condition.

15 · Functions

A function is a reusable block of code designed to perform a specific task.

  • Improves code organization
  • Reduces repetition
  • Makes programs easier to maintain
  • Allows code reuse

Functions have:

  • Function name
  • Parameters
  • Return value
  • Function body

16 · Arrays and Strings

An array stores multiple values of the same data type under one name.

  • One-dimensional arrays
  • Multi-dimensional arrays
  • Accessing array elements using indexes

A string is a sequence of characters used to store text.

17 · Debugging and Testing

Debugging is the process of finding and fixing errors in a program.

  • Syntax errors: Mistakes in programming language rules.
  • Runtime errors: Errors that occur while the program runs.
  • Logical errors: The program runs but gives wrong results.

18 · Programming Best Practices

  • Write clean and readable code.
  • Use meaningful variable names.
  • Comment important sections of code.
  • Test programs regularly.
  • Break large problems into smaller functions.
  • Follow consistent coding standards.

  • Flowcharts and pseudocode help design program logic.
  • C programming teaches fundamental programming concepts.
  • Variables store data while data types define the kind of data.
  • Control structures allow decisions and repetition.
  • Functions improve organization and code reuse.
  • Debugging helps create reliable software.
