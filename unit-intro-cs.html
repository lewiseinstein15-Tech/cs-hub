Introduction to Computer Science

The unit that hands you the vocabulary and mental model for everything else in this ecosystem. Read it like a story: computers evolving, ideas branching, and you finding where you fit in.

01 · The Big Idea

What is Computer Science?

Computer Science is the study of computation — how information is represented, processed, stored, and transmitted, and how we design step-by-step instructions (algorithms) to make machines solve problems for us.

It's less about "using computers" and more about the thinking behind them: how do you break a messy real-world problem into logical steps a machine can follow reliably, every time?

02 · Why It Matters

Why Study Computer Science?

Nearly every modern industry — banking, agriculture, healthcare, government, entertainment — now runs on software. Understanding computation means you can automate the tedious, analyze the complex, and build tools other people rely on.

In the Kenyan context specifically: mobile money systems, e-government portals, agri-tech platforms, and telecom infrastructure are all built and maintained by computer scientists — this is a field with direct, visible impact.

03 · History

Generations of Computers

Computing hardware evolved in five broad waves — each one smaller, faster, and more accessible than the last:

  • 1st Gen (1940s–50s): Vacuum tubes. Room-sized machines like ENIAC — huge power draw, huge heat.
  • 2nd Gen (1950s–60s): Transistors replaced vacuum tubes — smaller, faster, more reliable.
  • 3rd Gen (1960s–70s): Integrated circuits (ICs) packed multiple transistors onto one chip.
  • 4th Gen (1970s–present): Microprocessors — an entire CPU on a single chip. This is the era of the personal computer.
  • 5th Gen (present–future): AI-driven computing, parallel processing, and machine learning at the core of how systems are designed.
04 · Foundations

Hardware vs. Software

Hardware is everything you can physically touch: the CPU, RAM, storage drives, keyboard, screen. It's the "body" of the computer.

Software is the instructions telling that hardware what to do — the "mind." It splits into two types:

  • System software — operating systems (Windows, Linux, Android) that manage the hardware itself.
  • Application software — the programs you actually use: browsers, word processors, games, mobile apps.
05 · The Language Underneath

Number Systems

At the lowest level, computers only understand two states: on and off — represented as binary (base 2: 0s and 1s). Every photo, song, and keystroke is ultimately just patterns of 0s and 1s.

Because binary gets long and hard to read, computer scientists also work with:

  • Decimal (base 10) — the numbers you use daily.
  • Octal (base 8) — occasionally used in older systems.
  • Hexadecimal (base 16) — used constantly in programming to represent binary data compactly (e.g. colour codes, memory addresses).

You'll practice converting between these systems — it's a skill that shows up again in Discrete Structures and Computer Organization.

06 · Where You're Headed

Branches & Career Paths

Computer Science branches into specialized areas — and this hub's later zones map directly onto them:

  • Software Engineering — designing and building large-scale applications.
  • Artificial Intelligence & Machine Learning — systems that learn patterns from data.
  • Databases — organizing and retrieving information at scale.
  • Networks & Cybersecurity — connecting systems, and protecting them.
  • Human-Computer Interaction — designing how people actually use technology.

Career-wise, this feeds into roles like Software Engineer, Data Scientist, Cybersecurity Analyst, Cloud Engineer, and Systems Administrator — all covered later in this hub's Career Map.

07 · The Core Skill

Problem Solving & Algorithms

An algorithm is simply a precise, ordered set of steps to solve a problem — like a recipe. Before you write a single line of code, computer scientists are trained to:

  • Understand the problem clearly (inputs, outputs, constraints).
  • Break it into smaller sub-problems.
  • Design a step-by-step solution (often as pseudocode or a flowchart).
  • Only then translate that solution into a programming language.

This "think before you code" habit is what separates strong programmers from those who just memorize syntax — and it's exactly what Programming I builds on next.

08 · Clearing Up Confusion

CS vs. IT vs. Software Engineering

These terms get used interchangeably, but they're distinct:

  • Computer Science — the theory of computation: algorithms, data structures, why things work.
  • Information Technology (IT) — deploying, maintaining, and supporting computer systems and networks in an organization.
  • Software Engineering — applying engineering discipline (planning, testing, teamwork) to build large, reliable software systems.

A CS degree gives you the theoretical foundation that all three of these careers draw from.

📌 Quick Recap for Revision

  • CS = the study of computation and problem-solving via algorithms.
  • 5 generations of computers: vacuum tubes → transistors → ICs → microprocessors → AI-era computing.
  • Hardware = physical; Software = system vs. application.
  • Number systems: binary, decimal, octal, hexadecimal.
  • Branches: software engineering, AI/ML, databases, networks, HCI.
  • Algorithms = precise steps to solve a problem — think before you code.
  • CS ≠ IT ≠ Software Engineering — related but distinct.
